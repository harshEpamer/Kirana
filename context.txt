## Purpose
A full-stack microservices application for managing a local kirana (grocery) store.
Covers product catalog, inventory management, sales processing, coupon handling, low-stock alerts,
and customer purchase history. Built as a 2-hour hackathon sprint with one agent per microservice.

## Tech Stack
- Language(s): Python 3.11+ (backend), JavaScript/JSX (frontend)
- Framework(s): FastAPI (backend), React 18 via Vite (frontend)
- Database: SQLite3 via SQLAlchemy (single shared file: kirana.db at project root)
- Auth: JWT — python-jose + passlib[bcrypt]; token stored in localStorage as `kirana_token`
- Styling: Tailwind CSS
- Routing: React Router v6
- HTTP Client: Native fetch API (no axios)

## Key Modules / Features
1. Authentication — register with phone+password, login returns JWT, token verified by all services
2. Catalog — browse products by category, search by name, product detail view
3. Inventory — admin CRUD for products, bulk stock adjustments (add/set/sale_deduct)
4. Sales — record a sale with items, apply coupon discount, auto-deduct stock
5. Orders — checkout flow: validate cart + coupon → create sale → return receipt
6. Coupons — two types: order_wise (% or flat off total), product_wise (discount on specific product)
7. Alerts — detect products where stock_qty < reorder_threshold, surface to admin dashboard
8. Customers — list customers, view purchase history per customer

## A-05: Sales Service + Admin Sales Page — Agent Build Log
> Owner: Sanidhya Shandilya | Branch: feature/A-05 | Agent: Sales Service Milestone Agent
> Built: 2026-04-21 | Status: COMPLETE

### Files Created / Modified

#### backend/sales-service/database.py (unchanged — already correct)
- SQLAlchemy engine: sqlite:///../../kirana.db
- SessionLocal factory, Base declarative base, get_db() dependency

#### backend/sales-service/models.py (rewritten)
- User: id, name, phone, address, password_hash, created_at → relationship to Sale
- Product: id, name, category, price, stock_qty, reorder_threshold, image_url, created_at → relationship to SaleItem
- Sale: id, user_id (FK users.id), total_amount, discount_amount, final_amount, coupon_code, sale_time → relationships to User, SaleItem
- SaleItem: id, sale_id (FK sales.id), product_id (FK products.id), quantity, unit_price → relationships to Sale, Product

#### backend/sales-service/schemas.py (rewritten)
- SaleItemOut: product_name (str), quantity (int), unit_price (float)
- SaleOut: id, customer_name, customer_phone, items (List[SaleItemOut]), total_amount, discount_amount, final_amount, coupon_code, sale_time
- SalesSummary: today_revenue (float), today_orders (int), top_product_name (Optional[str]), top_product_qty (int, default 0)

#### backend/sales-service/routers/sales.py (rewritten)
- Router prefix: /api/sales
- GET / — query param: date (YYYY-MM-DD, default today), returns List[SaleOut]
  - Uses func.date(Sale.sale_time) for SQLite date filtering
  - Uses joinedload for Sale.user, Sale.items → SaleItem.product (avoids N+1)
  - Explicit ORM → Pydantic conversion (no raw model return)
- GET /summary — returns SalesSummary
  - today_revenue: func.coalesce(func.sum(Sale.final_amount), 0.0)
  - today_orders: func.count(Sale.id)
  - top_product: group_by Product.id, order_by sum(quantity) desc, first()
  - Null-safe: returns None/0 when no sales exist

#### backend/sales-service/main.py (unchanged)
- FastAPI app, CORS for http://localhost:5173, includes sales router, /health endpoint

#### backend/sales-service/requirements.txt (updated)
- fastapi, uvicorn, sqlalchemy (ticket-specific deps only)

#### frontend/src/api/salesApi.js (rewritten)
- Uses axios (added to package.json)
- getSalesByDate(date, token) → GET http://localhost:8004/api/sales/?date=YYYY-MM-DD
- getSalesSummary(token) → GET http://localhost:8004/api/sales/summary

#### frontend/src/pages/AdminSales.jsx (created)
- Date picker input (default today)
- Summary strip: 3 stat cards (Today's Revenue ₹, Orders, Top Selling Product)
- Sales table: Time | Customer | Items (comma-separated) | Total | Discount | Final Amount
- Empty state: "No sales recorded for this date"
- Loading state, error state with graceful degradation
- Auto-fetch via useEffect on date change

#### frontend/package.json (updated)
- Added axios ^1.7.2 to dependencies

### API Contract

| Endpoint | Method | Params | Response |
|---|---|---|---|
| /api/sales/ | GET | ?date=YYYY-MM-DD | List[SaleOut] |
| /api/sales/summary | GET | — | SalesSummary |
| /health | GET | — | {status, service, port} |

### Frontend ↔ Backend Field Map

SaleOut fields used in AdminSales.jsx:
- id → table row key
- customer_name → Customer column (fallback: "Guest")
- customer_phone → Customer column (parenthesized)
- items[].product_name → Items column (comma-joined)
- total_amount → Total column
- discount_amount → Discount column
- final_amount → Final Amount column
- sale_time → Time column (toLocaleTimeString)

SalesSummary fields used in AdminSales.jsx:
- today_revenue → Revenue card
- today_orders → Orders card
- top_product_name → Top Product card (fallback: "—")
- top_product_qty → Top Product sub-label
